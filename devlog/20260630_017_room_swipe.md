# 017. 프로그램 룸 전환 좌우 스와이프(모바일)

- 날짜: 2026-06-30
- 상태: 완료(테스트 서버 0.1.11 반영, git/이미지 푸시).

## 배경

발표 장소(룸)를 화면 위 탭 버튼으로만 전환하던 것을, 스마트폰에서 **좌우 스와이프**로도
전환할 수 있게 해달라는 요청.

## 한 일 (`program.html`)

- `.content`에 `touchstart`/`touchend`(passive) 핸들러 추가.
  - 가로 이동 60px 이상 + 세로보다 1.5배 우세할 때만 스와이프로 인정(세로 스크롤 오발화 방지).
  - 왼쪽으로 밀기(dx<0) → 다음 룸, 오른쪽 → 이전 룸. 기존 `activate(panel)` 재사용.
  - 양끝에서는 무시(순환 없음).
  - 선택 탭이 보이도록 **탭바만 가로 스크롤**(`bar.scrollTo`)해 페이지 세로 점프 방지.

## 검증

- 렌더 JS `node --check` 통과, touch 핸들러 존재 확인.
- 테스트 서버 v0.1.11 응답 확인.

## 메모

- 데스크톱/마우스는 기존 탭 클릭 그대로.
- `scrollIntoView(block:'nearest')`는 아래로 스크롤된 상태에서 페이지를 위로 끌어올릴 수
  있어 사용하지 않고, 탭바 컨테이너의 `scrollLeft`만 조정.
