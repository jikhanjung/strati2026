# 002. 초록집 파서 구현

- 날짜: 2026-06-29
- 상태: 완료
- 산출: `scripts/parse_abstracts.py` → `output/abstracts.json` (607건)

## 한 일

PyMuPDF span의 폰트/크기/위치(y)/flags 기반으로 초록집 923p를 구조화 파싱.

레코드 필드: `id, session, page, title, authors[{name, affil_refs, corresponding}],
affiliations_raw[], abstract, keywords_raw, keywords[]`.

## 해결한 파싱 함정

1. **러닝헤더/푸터**: y<82(헤더)·y>770의 9pt(푸터 페이지번호) 제거. 헤더에서 세션코드만 추출.
2. **세션 구분자 페이지**("Session GX" 26pt): 세션만 갱신하고 레코드에서 폐기.
3. **제목 내 작은 글리프로 제목이 쪼개짐**:
   - en-dash(`–`)가 11pt로 식자됨 (제목은 14pt) → 제목 분리.
   - 아래/위첨자(CO₂의 2, δ¹³C의 13, pCO₂ 등)가 9pt → 제목 분리.
   - 해결: 제목 단계에서 size<10.5 글리프는 제목에 흡수. 11pt 저자명에서 비로소 저자 단계 전환.
4. **일부 초록은 헤딩/본문이 9.5pt로 작게 식자**(예: G18 p522) → size 기반 skip을 폐기하고
   헤딩("Abstract"/"Key words")은 **볼드+텍스트**로만 인식.

- TOC 항목 수 vs 파싱 수: **30개 세션 중 28개 정확 일치**. 불일치 G1/S2/S11은 TOC 카운터
  (줄바꿈·숫자 포함 제목 오집계) 쪽 한계로 확인. G1은 수기 대조 결과 파서(10)가 정답.
- 중복 제목 0, 제목/저자/본문 누락 0.
- 키워드 없음 5건은 PDF에 실제로 "Key words" 줄이 없는 정상 케이스.
- 교신저자(`*`) 155/607 — 나머지는 원문에 `*` 표기가 없는 것(파서 정상).
- 본문 길이 중앙값 2065자(min 726), 저자 중앙값 4명.
