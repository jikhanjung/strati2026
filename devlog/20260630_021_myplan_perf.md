# 021. My Plan 로딩 속도 개선

- 날짜: 2026-06-30
- 상태: 완료(테스트 서버 0.1.14 반영 예정).

## 배경

My Plan을 띄울 때 느림. 원인: `/api/talks/`가 **전체 프로그램(457 talks, 원본 153KB)** 을
매번 무압축·무캐시로 내려주고 클라이언트가 북마크 id로 걸러냈음.

## 한 일

### 1) 북마크한 발표만 요청 (`views.py`, `timetable.html`)
- `api_talks`에 `?ids=` 지원: 주어지면 그 발표만 직렬화. 휴식은 정확도를 위해 전체에서
  도출하되, 반환된 발표 **직전 휴식만** 남김.
- My Plan fetch를 `?v={version}&ids=<북마크>`로 변경.
- 효과: 북마크 3개 기준 **156,760 → 968 바이트**.

### 2) gzip 압축 (`settings.py`)
- `django.middleware.gzip.GZipMiddleware` 추가. 전체 응답 기준 153KB → ~31KB.

### 3) 장기 캐싱 + 버전 무효화 (`views.py`, `timetable.html`)
- `api_talks` 응답 `Cache-Control: public, max-age=86400`.
- fetch URL에 `?v={{ version }}` → 같은 버전·같은 북마크면 브라우저 캐시에서 즉시,
  배포로 버전이 바뀌면 자동으로 새로 받음.

## 검증

- `?ids=10,11,12` → 968 bytes(talks 3, 직전 휴식만).
- gzip: Content-Encoding gzip, 31,101 bytes. Cache-Control 헤더 확인.
- `manage.py check` 통과.

## 메모

- ids 없이 호출하면 기존처럼 전체 반환(호환).
- 캐시 키가 (버전 + 북마크 id 집합)이라, 북마크를 바꾸면 URL이 바뀌어 자동 갱신.
