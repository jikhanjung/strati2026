# 033. 사진 동기화 (서버 저장 + 서명 URL)

- 날짜: 2026-06-30
- 상태: 완료(서버 end-to-end 검증). 운영은 nginx+https 전제.

## 설계

- 사진을 **토큰(버킷)별로 서버에 저장** → 링크된 기기들이 공유. 파일은 미디어 볼륨, DB엔 메타만.
- `<img>`가 헤더 인증 불가 → **서명 URL**(HMAC(SECRET, id:exp), 30일 만료)로 접근.
- 표시는 항상 서버 URL(브라우저 HTTP 캐시). IndexedDB는 **구버전 로컬 사진 이관용**으로만 남김.
- 쿼터(넉넉): 장당 8MB · talk당 50장 · 토큰당 1GB.

## 서버 (`models.py` SyncPhoto + migration 0006, `views.py`, `urls.py`, `settings.py`)

- `POST /api/photos/upload/` (헤더 `X-Device`, multipart `talk`,`file`): 쿼터 검사 후 저장.
- `GET /api/photos/?talk=ID`: 목록(서명 URL 포함).
- `GET /api/photos/talks/`: 사진 가진 talk id(카드 마커용).
- `GET /api/photos/file/<id>?exp&sig`: 서명 검증 후 스트리밍(FileResponse, private 캐시).
- `POST /api/photos/<id>/delete/`: 소유 토큰 확인 후 파일+행 삭제.
- `settings`: MEDIA_ROOT(`STRATI_MEDIA_ROOT`), 업로드 한도, 쿼터.

## 클라이언트 (`app.js`, `talk_detail.html`, `timetable.html`)

- 사진 계층을 서버 기반으로 교체: `addPhotoFile`(축소→업로드), `getPhotos`(목록), `deletePhoto`, `syncPhotoTalks`(마커).
- 로드 시 `syncPhotoTalks` + `migratePhotos`(구 IndexedDB 사진 1회 업로드). 페어링 후에도 마커 갱신.
- 상세/내 플랜의 썸네일·라이트박스를 `p.url`(서명 URL)로.

## 배포 (`deploy/host/docker-compose.yml`)

- 미디어 볼륨 마운트 `/srv/strati2026/media:/app/media` + `STRATI_MEDIA_ROOT=/app/media`.

## 검증

- 서버: 업로드→목록→talks→서명URL 다운로드(200)→잘못된 sig 403→삭제(파일정리) 전부 정상.
- `node --check`, `manage.py check` 통과.

## 메모/한계

- 사진은 토큰 버킷 소유 → 솔로 기기도 서버 백업됨(브라우저 삭제 후에도 보존). 페어링하면 공유.
- 오프라인 추가는 현재 미지원(업로드 큐는 후속). 추가 시 온라인 필요.
- 운영 디스크/백업 관리 대상(미디어 볼륨). 비활성 토큰 정리는 후속.
- 기기 unlink(토큰 회전) 시 그 기기 사진은 옛 토큰에 남음(엣지).
