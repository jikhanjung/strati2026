# 022. My Plan 절대 시간 기반 + 빈 시간 압축 + 카드 높이 확대

- 날짜: 2026-06-30
- 상태: 완료(테스트 서버 0.1.15 반영 예정).

## 배경

020의 겹침 컬럼 그리드가 분당 3.4px라 15분 발표 카드가 ~50px로 너무 작아 정보가 잘림.
"My Plan 전체를 절대 시간 기반으로 + 카드 정보가 웬만하면 다 보이게" 요청.
북마크만 보이는 특성상 절대 시간이면 빈 시간이 거대 여백이 되는데, 사용자 선택 = **빈 시간 압축**.

## 발표 길이 분포(참고)

15분 377 · 20분 58 · 30분 9 · 70~100분 11(plenary). → 15분이 압도적이라 스케일을 키움.

## 한 일 (`timetable.html`, `style.css`)

- 분당 **7px**(PX). 15분≈105px → 시간·제목·세션·저자·장소 다 보임. MIN_CARD 96.
- **단독 발표**: 전체폭, 비례 높이를 `min-height`로(잘림 방지, 내용 길면 늘어남).
  과도한 여백 방지로 상한 `SINGLE_MAX=180`(plenary 90분도 180px).
- **겹침**: 비례 컬럼 그리드 유지 — 컬럼 폭 86%(옆 카드 살짝 보임) + 좌우 스크롤.
- **블록 사이 빈 시간**: 시간 비례 대신 **압축**.
  - 휴식이면 `☕/🍽️ Break · HH:MM–HH:MM` 라벨,
  - 아니면 `··· Xh Ym ···` (점선 + 중앙 라벨),
  - 12분 미만은 얇은 스페이서.
- 현재 시각 줄(`nowY`)은 rect 기반이라 이 레이아웃에서도 그대로 동작(갭에는 data-start 없음).

## 검증

- 렌더 JS `node --check` 통과.
- 레이아웃 단위 시뮬: 겹침 [A,B] 컨테이너 111px·카드 ≥96px, 빈 시간 "2h 30m"/"2h" 압축,
  단독 15분 105px, plenary 90분 180px 상한.

## 메모

- 절대 시간은 "발표/겹침 구간 내"에서 비례, 블록 사이 빈 시간은 압축(사용자 선택).
- 15분 미만 발표는 없음(최소 15분)이라 MIN_CARD floor가 슬롯을 침범하지 않음.
